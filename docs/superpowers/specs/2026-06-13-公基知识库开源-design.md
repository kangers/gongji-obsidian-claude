# 公基知识库开源 — 设计规格说明

> 创建时间: 2026-06-13 | 状态: 待审阅 | 作者: kanger

---

## 一、背景与目标

### 1.1 背景
用户使用 LLM Wiki 方法论构建了一个公共基础知识（公基）备考 Obsidian 知识库，经过实际考试验证复习效果良好。现决定将其开源，供所有公基备考者使用。

### 1.2 目标
1. **发布**：将知识库发布到 GitHub，面向通用公基备考者
2. **隐私**：彻底删除所有个人信息、军事身份信息、做题记录
3. **双模**：同时服务 Obsidian 用户（clone 即用）和普通浏览器用户（GitHub Pages）
4. **可维护**：建立社区贡献基础设施，支持持续维护

### 1.3 非目标（一期不做）
- PDF 教材的 OCR 提取（二期 Roadmap）
- 多语言支持
- 移动端 App

---

## 二、核心决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 发布模式 | 双模（Obsidian 源码 + GitHub Pages 文档站） | 最大化受众，同时保持 Obsidian 原生体验 |
| 许可证 | CC BY-SA 4.0 | Wikipedia 同款，保护知识共享的衍生自由 |
| 目标受众 | 通用公基备考者（不限于军转） | 扩大受益面 |
| 军事信息 | 完全去军事化 | 用户明确要求，避免关联个人军事身份 |
| 做题记录 | 全部删除/改写 | 用户明确要求 |
| PDF 处理 | 一期完全排除，二期 OCR | 版权安全优先 |
| 维护策略 | 用户持续维护 + 社区 PR | 保证内容质量与时效性 |

#### 实体页面处理
- `wiki/实体/公务员法.md` → 删除文件（通用公基不含此科目）
- `wiki/实体/监察法.md` → 删除文件（同上）
- `wiki/来源/公共基础知识考试政策.md` → 改写为通用考试政策来源页
- `wiki/index.md` 中相关条目 → 删除

#### 政策解读目录处理
- `政策解读/2026年关于做好2026年全省考生考试工作的函（考试管理机构）白头.md` → 删除或归档
- `政策解读/2026年公共基础知识考试系统备考指南.md` → 改写为通用备考指南
- `政策解读/` 下的 PDF 文件 → 不提交

#### 模拟卷题目中的军事/军人表述
模拟卷题目中以下内容删除/改写（完全去军事化）：
- 题目标题/引言中的"军转""劳动者"字样 → 改为中性表述
- "附加考点"标记 → 删除
- 涉及「公务员法」「监察法」等军转专属法律的选择题/判断题 → 逐题评估：通用公基也考则保留并泛化；纯军转专属则删除
- "服役""部队""军功""军衔"等军事场景描述 → 改写为中性场景或删除
- 时政类题目中「劳动者事务部」「考试管理机构」等机构 → 删除或替换为通用机构

注意：泛化不等于完全删除知识点。例如"公务员法"中的基本原则（公平、公开）是通用法理，可保留概念但去掉军转标签。

## 五、CI/CD 方案

### 5.1 GitHub Actions Workflow

触发条件：每次 push 到 `main` 分支

流程：
1. Checkout 代码
2. 设置 Python 环境
3. 运行链接转换脚本（`[[link]]` → `[text](path.md)` 到临时目录）
4. 安装 MkDocs Material + 插件
5. `mkdocs build` → `site/`
6. 部署到 GitHub Pages（`gh-pages` 分支）

### 5.2 链接转换脚本
- 语言：Python
- 输入：`wiki/`、`辅导班题目/` 等所有 `.md` 文件
- 逻辑：
  - `[[wiki/概念/政治.md|显示名]]` → `[显示名](wiki/概念/政治.md)`
  - `[[wiki/概念/政治.md]]` → `[政治](wiki/概念/政治.md)`
  - `[[笔记名]]` → `[笔记名](笔记名.md)`（同目录相对路径）
- 保留 YAML frontmatter（MkDocs 的 `mkdocs-material` 主题支持）

### 5.3 MkDocs 配置
- 主题：`material`
- 特性：搜索、暗色模式、导航折叠、代码高亮
- 插件：`search`（内置）、`awesome-pages`（自动导航）
- 域名：`<username>.github.io/gongji-wiki`（或自定义域名）

## 第一步：安装 Obsidian（你的知识管理软件）

### 什么是 Obsidian？
一个免费的笔记软件（类似 Notion，但更强大）。知识库文件就是
普通的 Markdown 文本，可以永久保存、自由迁移。

### 下载与安装

#### macOS
1. 打开 https://obsidian.md/download
2. 点击 "Download for macOS"（dmg 文件）
3. 双击下载的 .dmg，把 Obsidian 图标拖到 Applications 文件夹
4. 在启动台找到 Obsidian 并打开
5. 如果提示"无法验证开发者"：系统设置 → 隐私与安全性 → 仍要打开

#### Windows
1. 打开 https://obsidian.md/download
2. 点击 "Download for Windows"（.exe 文件）
3. 运行安装程序，一路"下一步"即可
4. 桌面双击 Obsidian 图标启动

#### Linux
1. 打开 https://obsidian.md/download
2. 下载 AppImage 文件
3. 右键 → 属性 → 权限 → 勾选"允许作为程序执行"
4. 双击运行

#### iPhone/iPad
1. App Store 搜索 "Obsidian" 安装
2. 免费，无内购

#### Android
1. Google Play 搜索 "Obsidian" 安装
2. 或从 https://obsidian.md/download 下载 APK

### 首次启动
1. 先关闭 Obsidian（后面会重新打开）
2. 暂时不需要创建 Vault，等知识库下载好再说

## 第四步：安装 Claudian 插件（让 Claude AI 接入 Obsidian）

> Claudian 是一个 Obsidian 插件，把 Claude AI 直接嵌入 Obsidian。
> 你可以在 Obsidian 里和 Claude 对话，让它帮你查询知识库、
> 生成模拟卷、解释错题。

### 4.1 开启社区插件

1. Obsidian → 左下角齿轮按钮 → Settings（设置）
2. 左侧菜单：Community plugins（社区插件）
3. 如果看到"Safe mode"提示：点 "Turn on community plugins"（开启社区插件）

### 4.2 浏览并安装 Claudian

1. 在 Community plugins 页面，点 "Browse"（浏览）
2. 搜索框输入：`Claudian`
3. 找到 "Claudian" 插件 → 点 "Install"（安装）
4. 安装完成后点 "Enable"（启用）

### 4.3 配置 Claudian 侧边栏

1. Obsidian 左侧竖条区域 → 右键 → 或点右上角三个点的菜单
2. 找到 Claudian 的图标（一个聊天气泡或机器人图标）
3. 点击打开 Claudian 对话面板
4. 现在右侧应该出现了一个聊天界面

## 第五步：获取 Claude API Key（让 AI 真正工作）

> Claude 是 Anthropic 公司开发的 AI 助手。你需要一个 API Key
> （类似"通行证"）来使用它。

### 5.1 注册 Anthropic 账号

1. 打开 https://console.anthropic.com
2. 点击 "Sign Up" → 用 Google 账号或邮箱注册
3. 验证邮箱（去收件箱点链接）
4. 登录后进入 Anthropic Console

### 5.2 充值（首次有免费额度）

1. Console 左侧菜单 → Billing（账单）
2. 首次注册通常赠送 $5 或更多免费额度
3. 如果额度用完需要充值：
   - 点击 "Add credits"
   - 支持 Visa/Mastercard 信用卡
   - 最低充值 $5（约 35 元人民币）
   - **一个月学习公基，$5 完全够用**

### 5.3 创建 API Key

1. Console 左侧菜单 → API Keys
2. 点击 "Create Key"
3. 给 Key 取个名字（比如"公基学习"）→ Create
4. **重要**：复制生成的 Key（以 `sk-ant-` 开头）
5. **警告**：这个 Key 只显示一次！立刻粘贴到下一步。

| 设置项 | 推荐值 | 说明 |
|--------|--------|------|
| Model | Claude Sonnet 4 | 性价比最高，够快够聪明 |

## 第七步：安装 Andrej Karpathy 的 LLM Wiki 技能

> Andrej Karpathy（前特斯拉 AI 总监、前 OpenAI 创始成员）
> 提出了 "LLM Wiki" 方法论。Claudian 支持加载他设计的
> 技能文件，让 Claude 更懂如何维护知识库。

### 7.1 理解技能文件的位置

技能文件在本知识库中已经包含（`.claude/skills/` 目录下）。
这些是 Karpathy 预定义的智能体配置，包括：
- `karpathy-guidelines` — LLM Wiki 维护指南
- 其他辅助技能

> ⚠️ 注意：由于 `.claude/` 目录在 git clone 时通常被
> `.gitignore` 排除，你可能看不到。我们会在 Step 7.3 处理。

### 7.2 在 Claudian 中加载技能

1. Obsidian → Settings → Claudian
2. 找到 "Skills" 或 "Agent" 相关设置
3. 设置技能搜索路径为 Vault 根目录
4. 保存设置

### 7.3 如果技能文件夹不存在（手动创建）

如果 git clone 后没有 `.claude/skills/` 目录，你需要手动创建。
打开终端（在 gongji-wiki 目录下），粘贴：

```bash
mkdir -p .claude/skills/karpathy-guidelines
```

然后从 Karpathy 的 GitHub 仓库下载技能文件：

```bash
# 下载 Andrej Karpathy 的 LLM Wiki 技能
curl -o .claude/skills/karpathy-guidelines/SKILL.md \
  https://raw.githubusercontent.com/karpathy/llm-wiki/main/.claude/SKILL.md
```

### 7.4 验证技能加载

1. 在 Claudian 对话框中输入："/skills"
2. 如果看到 `karpathy-guidelines` 在列表中 → ✅ 技能加载成功！
3. 后续和 Claude 对话时，可以输入 `/karpathy-guidelines` 来激活这个技能

## 常见问题 FAQ

### Q: 这个知识库要钱吗？
A: 知识库完全免费（CC BY-SA 4.0 开源协议）。
   Obsidian 个人使用免费。
   Claude API 按量付费，首次送 $5 额度，一个月备考约花 $2-5。

### Q: 我需要编程基础吗？
A: 不需要。跟着这篇指南一步一步做就行。
   有 Git 基础会很方便（更新知识库），但不是必需的。

### Q: Claude 的回答准确吗？
A: Claude 的知识库内容经过人工整理和考试验证，
   比直接问通用 AI 准确得多。但最终判断请以教材为准。

### Q: 我的数据会泄露吗？
A: 不会。API Key 只存在于你的电脑上。
   向 Claude 提问时，只发送你当前问题涉及的文件内容。

### Q: 手机能用吗？
A: 可以！Obsidian 有 iOS/Android 版，Claudian 也支持移动端。
   但 API 配置在手机上步骤略多，建议先电脑配置好再同步到手机。

### Q: 如何获取知识库更新？
A: 如果用 git clone，运行 `git pull` 即可。
   如果下载的 ZIP，重新下载覆盖即可（注意备份自己的笔记）。
